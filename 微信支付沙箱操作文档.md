# 微信支付沙箱（Sandbox）操作文档

> 适用于本项目「电影票预约系统」的支付模块测试。沙箱环境无真实资金流动，所有交易为模拟数据。

---

## 一、沙箱环境是什么

微信支付提供的仿真测试系统，与生产环境**完全隔离**：
- 用户不会被真实扣款
- 商户不会有资金入账
- 所有订单、退款均为假数据
- 适合开发阶段联调与验收

---

## 二、前置条件

| 条件 | 说明 |
|------|------|
| 微信商户号 | 在 [pay.weixin.qq.com](https://pay.weixin.qq.com) 注册并开通 |
| API 密钥（V2） | 商户平台 → 账户中心 → API安全 → 设置 API 密钥（32位） |
| 公众号 / 小程序 AppID | 绑定到该商户号 |

---

## 三、获取沙箱密钥（sandbox_signkey）

沙箱环境使用**独立密钥**签名，有效期 3 天，到期需重新获取。

### 请求

```
POST https://api.mch.weixin.qq.com/xdc/apiv2getsignkey/sign/getsignkey
Content-Type: application/xml
```

### 请求体（XML）

```xml
<xml>
  <mch_id>你的商户号</mch_id>
  <nonce_str>随机字符串(≤32位)</nonce_str>
  <sign>用正式API密钥生成的MD5签名(大写)</sign>
</xml>
```

### 签名生成规则

1. 将参数按字母排序拼接：`mch_id=xxx&nonce_str=xxx`
2. 末尾追加 `&key=你的正式API密钥`
3. 对整个字符串做 MD5，转**大写**

### 返回示例

```xml
<xml>
  <return_code>SUCCESS</return_code>
  <sandbox_signkey>xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx</sandbox_signkey>
</xml>
```

> 后续所有沙箱请求，sign 均用此 `sandbox_signkey` 生成，不再用正式密钥。

## 四、沙箱 API 调用方式

只需将正式接口 URL 加上沙箱路径前缀：

| 接口 | 正式地址 | 沙箱地址 |
|------|----------|----------|
| 统一下单 | `https://api.mch.weixin.qq.com/pay/unifiedorder` | `https://api.mch.weixin.qq.com/xdc/apiv2sandbox/pay/unifiedorder` |
| 查询订单 | `https://api.mch.weixin.qq.com/pay/orderquery` | `https://api.mch.weixin.qq.com/xdc/apiv2sandbox/pay/orderquery` |
| 申请退款 | `https://api.mch.weixin.qq.com/secapi/pay/refund` | `https://api.mch.weixin.qq.com/xdc/apiv2sandbox/secapi/pay/refund` |
| 关闭订单 | `https://api.mch.weixin.qq.com/pay/closeorder` | `https://api.mch.weixin.qq.com/xdc/apiv2sandbox/pay/closeorder` |

## 五、统一下单（沙箱示例）

### 请求体

```xml
<xml>
  <appid>你的AppID</appid>
  <mch_id>你的商户号</mch_id>
  <nonce_str>随机字符串</nonce_str>
  <sign>用sandbox_signkey生成的MD5签名(大写)</sign>
  <body>电影票-复仇者联盟4</body>
  <out_trade_no>自定义商户订单号</out_trade_no>
  <total_fee>101</total_fee>
  <spbill_create_ip>客户端IP</spbill_create_ip>
  <notify_url>https://你的域名/pay/notify</notify_url>
  <trade_type>NATIVE</trade_type>
</xml>
```

### 重要：沙箱金额规则

沙箱环境**仅支持特定测试金额**，不可随意传值。常见验收用例金额：

| 用例场景 | total_fee（单位：分） |
|----------|----------------------|
| 正常支付 | 101 |
| 付款码支付 | 101 |
| 退款（全额） | 101 |
| 退款（部分） | 102 |
| 订单金额为0（异常） | 0 |

> 具体用例金额以微信官方「验收用例」为准，可在商户平台查看。

## 六、验收流程总结

```
┌─────────────────────────────────────────────────────┐
│  1. 用正式 API 密钥 → 获取 sandbox_signkey          │
│  2. 替换接口 URL（加 /xdc/apiv2sandbox/ 前缀）      │
│  3. 所有 sign 改用 sandbox_signkey 生成             │
│  4. 按指定金额传参，执行用例                         │
│  5. 验证返回值是否符合预期                           │
│  6. 全部用例通过 → 可切换到正式环境上线              │
└─────────────────────────────────────────────────────┘
```

## 七、Java 代码示例（获取沙箱密钥）

以下示例可直接集成到本项目中（需引入 HTTP 库如 OkHttp / HttpURLConnection）：

```java
import java.io.*;
import java.net.*;
import java.security.MessageDigest;
import java.util.UUID;

public class WechatSandbox {

    private static final String MCH_ID = "你的商户号";
    private static final String API_KEY = "你的正式API密钥(32位)";

    public static void main(String[] args) throws Exception {
        String nonceStr = UUID.randomUUID().toString().replace("-", "").substring(0, 32);

        // 拼接签名字符串
        String signStr = "mch_id=" + MCH_ID + "&nonce_str=" + nonceStr + "&key=" + API_KEY;
        String sign = md5(signStr).toUpperCase();

        // 构造 XML
        String xml = "<xml>"
                + "<mch_id>" + MCH_ID + "</mch_id>"
                + "<nonce_str>" + nonceStr + "</nonce_str>"
                + "<sign>" + sign + "</sign>"
                + "</xml>";

        // 发送请求
        String url = "https://api.mch.weixin.qq.com/xdc/apiv2getsignkey/sign/getsignkey";
        String response = postXml(url, xml);
        System.out.println("沙箱密钥响应：\n" + response);
    }

    private static String md5(String input) throws Exception {
        MessageDigest md = MessageDigest.getInstance("MD5");
        byte[] digest = md.digest(input.getBytes("UTF-8"));
        StringBuilder sb = new StringBuilder();
        for (byte b : digest) {
            sb.append(String.format("%02x", b));
        }
        return sb.toString();
    }

    private static String postXml(String urlStr, String xml) throws Exception {
        URL url = new URL(urlStr);
        HttpURLConnection conn = (HttpURLConnection) url.openConnection();
        conn.setRequestMethod("POST");
        conn.setDoOutput(true);
        conn.setRequestProperty("Content-Type", "application/xml; charset=UTF-8");
        try (OutputStream os = conn.getOutputStream()) {
            os.write(xml.getBytes("UTF-8"));
        }
        try (BufferedReader br = new BufferedReader(new InputStreamReader(conn.getInputStream(), "UTF-8"))) {
            StringBuilder sb = new StringBuilder();
            String line;
            while ((line = br.readLine()) != null) sb.append(line).append("\n");
            return sb.toString();
        }
    }
}
```

## 八、注意事项

1. **签名必须大写** — 沙箱环境要求 sign 值为大写 MD5，否则返回签名错误
2. **请求必须 XML 格式** — 不支持 JSON，否则返回 HTTP 204
3. **沙箱密钥有效期 3 天** — 过期后需重新获取
4. **查单只支持 out_trade_no** — 沙箱不支持用微信订单号查询
5. **异常测试用例** — 需在 HTTP Header 中添加：`Wechatpay-Negative-Test: {用例名}`
6. **不要在沙箱环境做性能/压力测试** — 仅用于功能验收
