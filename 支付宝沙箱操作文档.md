# 支付宝沙箱（Sandbox）操作文档

> 适用于本项目「电影票预约系统」的支付模块测试。沙箱环境无真实资金流动，所有交易为模拟数据。

---

## 一、沙箱环境是什么

支付宝提供的仿真测试系统，与生产环境**完全隔离**：
- 用户不会被真实扣款
- 商户不会有资金入账
- 所有订单、退款均为假数据
- 使用沙箱版支付宝钱包扫码/付款
- 适合开发阶段联调与验收

---

## 二、前置条件

| 条件 | 说明 |
|------|------|
| 支付宝开放平台账号 | 在 [open.alipay.com](https://open.alipay.com) 注册并完成实名认证 |
| 应用 AppID | 在开放平台 → 开发者中心 → 创建应用，获取应用的 AppID |
| 沙箱环境 | 在开发者中心 → 沙箱环境，系统会自动生成沙箱账号及应用 |
| 商户 PID | 沙箱环境页面的「商户UID」 |
| 应用私钥 & 支付宝公钥 | 沙箱环境下自行生成 RSA2 密钥对，或使用支付宝提供的工具生成 |

---

## 三、进入沙箱环境

1. 登录 [open.alipay.com](https://open.alipay.com)
2. 进入「开发者中心」→「沙箱环境」
3. 系统自动为你生成：

| 项目 | 说明 |
|------|------|
| 沙箱 AppID | 替换生产 AppID 使用 |
| 沙箱PID（商户UID） | 2088 开头的一串数字 |
| 沙箱网关 | `https://openapi-sandbox.dl.alipaydev.com/gateway.do` |
| 沙箱买家账号 | 用于测试支付的买家支付宝账号 |
| 沙箱买家登录密码 | 买家账号登录密码（默认 111111） |
| 沙箱买家支付密码 | 支付时输入的密码（默认 111111） |
| 沙箱钱包 App | 下载专用沙箱支付宝钱包扫码支付 |

## 四、密钥配置

支付宝使用 **RSA2（SHA256WithRSA）** 签名，与微信支付的 MD5 不同。

### 4.1 生成密钥对

使用支付宝开放平台助手（或 OpenSSL）生成：

```bash
# 生成 2048 位 RSA 私钥
openssl genrsa -out app_private_key.pem 2048

# 从私钥导出公钥
openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem
```

### 4.2 配置到沙箱环境

1. 将**应用公钥**内容粘贴到沙箱环境的「应用公钥」栏
2. 支付宝会自动生成对应的「支付宝公钥」，保存下来
3. 或者将公钥粘贴后选择「上传公钥证书」方式（推荐生产环境使用）

| 密钥 | 用途 | 谁持有 |
|------|------|--------|
| 应用私钥 | 对请求做签名 | 开发者持有 |
| 支付宝公钥 | 验签支付宝的响应 | 开发者持有 |

## 五、沙箱 API 调用方式

### 5.1 网关地址

| 环境 | 网关地址 |
|------|----------|
| 沙箱 | `https://openapi-sandbox.dl.alipaydev.com/gateway.do` |
| 生产 | `https://openapi.alipay.com/gateway.do` |

只需要替换网关地址，其他参数一致。

### 5.2 公共请求参数

所有接口使用 **POST + JSON / URL-encoded** 方式调用：

```
POST https://openapi-sandbox.dl.alipaydev.com/gateway.do
Content-Type: application/x-www-form-urlencoded; charset=utf-8
```

| 参数 | 说明 | 示例 |
|------|------|------|
| app_id | 沙箱 AppID | 20210001XXXXXXXX |
| method | 接口名称 | alipay.trade.precreate |
| charset | 编码 | utf-8 |
| sign_type | 签名算法 | RSA2 |
| sign | 签名 | 用应用私钥对排序后的请求参数签名 |
| timestamp | 时间戳 | 2024-01-01 12:00:00 |
| version | API 版本 | 1.0 |
| biz_content | 业务参数 JSON 字符串 | `{"out_trade_no":"...","total_amount":"...",...}` |

### 5.3 接口列表（常用）

| 场景 | 接口方法 | 说明 |
|------|----------|------|
| PC 网页支付 | alipay.trade.page.pay | 跳转支付宝收银台 |
| 扫码支付（主扫） | alipay.trade.precreate | 生成支付二维码 |
| 条码支付（被扫） | alipay.trade.pay | 扫描用户付款码 |
| 订单查询 | alipay.trade.query | 按订单号查询支付状态 |
| 退款 | alipay.trade.refund | 全额或部分退款 |
| 退款查询 | alipay.trade.fastpay.refund.query | 查询退款结果 |
| 订单关闭 | alipay.trade.close | 关闭未支付的订单 |

## 六、扫码下单示例（alipay.trade.precreate）

### 请求体（URL-encoded 整体）

```
app_id=20210001XXXXXXXX
method=alipay.trade.precreate
charset=utf-8
sign_type=RSA2
timestamp=2024-01-01+12%3A00%3A00
version=1.0
notify_url=https%3A%2F%2F你的域名%2Fpay%2Fnotify
biz_content=%7B%22out_trade_no%22%3A%22MOVIE20240101000001%22%2C%22total_amount%22%3A%2249.90%22%2C%22subject%22%3A%22%E7%94%B5%E5%BD%B1%E7%A5%A8-%E5%A4%8D%E4%BB%87%E8%80%85%E8%81%94%E7%9B%9F4%22%2C%22body%22%3A%22IMAX+3D+%E5%85%91%E6%8D%A2%E5%88%B8%22%7D
sign=Base64(应用私钥签名)
```

### biz_content（JSON 格式）

```json
{
  "out_trade_no": "MOVIE20240101000001",
  "total_amount": "49.90",
  "subject": "电影票-复仇者联盟4",
  "body": "IMAX 3D 兑换券",
  "timeout_express": "15m"
}
```

### 注意：沙箱实际扣款

沙箱环境下，订单**不会**真实扣款，但沙箱钱包余额会显示扣减。每次进入沙箱钱包，余额会被重置为 **100,000 元**，可放心测试。

### 返回示例

```json
{
  "alipay_trade_precreate_response": {
    "code": "10000",
    "msg": "Success",
    "out_trade_no": "MOVIE20240101000001",
    "qr_code": "https://qr.alipay.com/bax03431xxxxxxxxxxxxxxx"
  },
  "sign": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}
```

- `code=10000` 表示调用成功
- `qr_code` 为二维码链接，可生成二维码后用沙箱钱包扫码支付

## 七、验收流程总结

```
┌──────────────────────────────────────────────────────┐
│  1. 在开放平台进入沙箱环境                            │
│  2. 获取沙箱 AppID、PID、网关地址、沙箱买家账号       │
│  3. 生成 RSA2 密钥对，配置应用公钥                    │
│  4. 替换网关为沙箱地址                                │
│  5. 用沙箱钱包扫码支付或跳转收银台                    │
│  6. 验证同步返回 + 异步通知是否符合预期               │
│  7. 全部用例通过 → 替换为生产网关和 AppID 上线        │
└──────────────────────────────────────────────────────┘
```

## 八、Java 代码示例（扫码下单）

以下示例可直接集成到本项目中（需引入支付宝 SDK）：

### 8.1 Maven 依赖

```xml
<dependency>
  <groupId>com.alipay.sdk</groupId>
  <artifactId>alipay-sdk-java</artifactId>
  <version>4.38.157.ALL</version>
</dependency>
```

### 8.2 配置类

```java
import com.alipay.api.AlipayConfig;

public class AlipaySandboxConfig {

    // 沙箱网关
    public static final String SERVER_URL = "https://openapi-sandbox.dl.alipaydev.com/gateway.do";
    // 沙箱 AppID
    public static final String APP_ID = "你的沙箱AppID";
    // 应用私钥（去掉头尾的 -----BEGIN/END RSA PRIVATE KEY-----）
    public static final String APP_PRIVATE_KEY = "你的应用私钥";
    // 支付宝公钥
    public static final String ALIPAY_PUBLIC_KEY = "你的支付宝公钥";

    public static AlipayConfig getConfig() {
        AlipayConfig config = new AlipayConfig();
        config.setServerUrl(SERVER_URL);
        config.setAppId(APP_ID);
        config.setPrivateKey(APP_PRIVATE_KEY);
        config.setAlipayPublicKey(ALIPAY_PUBLIC_KEY);
        config.setFormat("json");
        config.setCharset("UTF-8");
        config.setSignType("RSA2");
        return config;
    }
}
```

### 8.3 扫码下单服务

```java
import com.alipay.api.AlipayApiException;
import com.alipay.api.AlipayClient;
import com.alipay.api.DefaultAlipayClient;
import com.alipay.api.request.AlipayTradePrecreateRequest;
import com.alipay.api.response.AlipayTradePrecreateResponse;
import com.alipay.api.domain.AlipayTradePrecreateModel;
import com.alipay.api.AlipayConfig;

public class AlipaySandboxService {

    public String createQrCode(String outTradeNo, String subject, String amount) {
        AlipayConfig config = AlipaySandboxConfig.getConfig();
        AlipayClient alipayClient = new DefaultAlipayClient(config);

        AlipayTradePrecreateRequest request = new AlipayTradePrecreateRequest();
        AlipayTradePrecreateModel model = new AlipayTradePrecreateModel();
        model.setOutTradeNo(outTradeNo);
        model.setSubject(subject);
        model.setTotalAmount(amount);
        model.setTimeoutExpress("15m");
        request.setBizModel(model);
        // 异步通知地址
        request.setNotifyUrl("https://你的域名/pay/alipay/notify");

        try {
            AlipayTradePrecreateResponse response = alipayClient.execute(request);
            if (response.isSuccess()) {
                System.out.println("下单成功，二维码链接: " + response.getQrCode());
                return response.getQrCode();
            } else {
                System.out.println("下单失败: " + response.getCode() + " - " + response.getMsg());
                return null;
            }
        } catch (AlipayApiException e) {
            e.printStackTrace();
            return null;
        }
    }
}
```

### 8.4 异步通知验签

```java
import com.alipay.api.AlipayApiException;
import com.alipay.api.internal.util.AlipaySignature;
import java.util.Map;

public class AlipayNotifyHandler {

    public boolean verify(Map<String, String> params) {
        try {
            return AlipaySignature.rsaCheckV1(
                params,
                AlipaySandboxConfig.ALIPAY_PUBLIC_KEY,
                "UTF-8",
                "RSA2"
            );
        } catch (AlipayApiException e) {
            e.printStackTrace();
            return false;
        }
    }
}
```

## 九、注意事项

1. **网关地址必须替换** — 沙箱和生产仅网关地址不同，切换时务必检查
2. **签名算法为 RSA2** — 默认 SHA256WithRSA，与微信支付的 MD5 完全不同
3. **金额单位为元** — 支付宝接口的 `total_amount` 以元为单位（如 `49.90`），而非分，注意与微信支付区分
4. **沙箱钱包需单独下载** — 不能使用正式版支付宝扫码，必须下载沙箱版钱包 APK
5. **沙箱账号余额** — 每次登录沙箱钱包，余额自动重置为 100,000 元
6. **异步通知须公网可达** — 支付宝的异步通知需要公网 IP 或使用内网穿透工具（如 ngrok）
7. **不要在沙箱做性能/压力测试** — 沙箱环境仅用于功能验证
8. **秘钥安全** — 应用私钥不能泄露，生产环境建议使用公钥证书模式
9. **biz_content 需 URL 编码** — 在构造请求参数时，biz_content 的值需要做一次 URL Encode
